# โปรแกรมเปรียบเทียบประสิทธิภาพการดึงข้อมูลสภาพอากาศด้วย Threading, Asyncio และ Process Pool

## บทนำ
โปรเจกต์นี้จัดทำขึ้นเพื่อศึกษาและเปรียบเทียบประสิทธิภาพการทำงานแบบ Concurrency ในภาษา Python ผ่านเทคนิค 3 รูปแบบ ได้แก่ Threading, Asyncio และ Multiprocessing (Process Pool) โดยใช้โจทย์ปัญหาที่ต้องมีการรอคอยระบบเครือข่าย (I/O Bound) เพื่อให้เห็นความแตกต่างของระยะเวลาในการประมวลผลเมื่อเทียบกับการทำงานแบบเรียงลำดับปกติ (Synchronous)

## ไอเดียนี้ทำอะไร และได้อะไร?
**สิ่งที่ทำ:** โปรแกรมจะทำการดึงข้อมูลอุณหภูมิปัจจุบันของเมืองต่างๆ ทั่วโลกจำนวน 20 เมือง ผ่านบริการ API จริงของเว็บไซต์ Open-Meteo (https://open-meteo.com) 
**สิ่งที่ได้:** 1. ได้เรียนรู้วิธีการเขียนโค้ดดึงข้อมูลผ่าน HTTP Request
2. ได้เห็นภาพความแตกต่างของระยะเวลาในการทำงาน (Execution Time) ระหว่างการเขียนโปรแกรมแบบปกติ กับการใช้ Threading, Asyncio และ Process Pool


## ทำไมถึงเลือกทำเรื่องนี้?
การดึงข้อมูลจากเว็บไซต์หรือ API (Network Request) เป็นลักษณะงานแบบ I/O Bound ซึ่งโปรแกรมมักจะต้องเสียเวลาไปกับการ "รอ" การตอบกลับจากเซิร์ฟเวอร์ การใช้โจทย์นี้จะช่วยดึงศักยภาพของ Threading และ Asyncio ออกมาให้เห็นได้อย่างชัดเจนที่สุดด้วยโค้ดที่สั้นและเข้าใจง่าย (Simple code)

## วิธีการทำงานของโปรแกรม
1. กำหนดพิกัด (Latitude, Longitude) ของเมืองเป้าหมายจำนวน 20 เมือง
2. ใช้เครื่องมือ 3 แบบ (Thread, Asyncio, Process Pool) เพื่อส่งคำขอไปดึงข้อมูลอุณหภูมิของทั้ง 20 เมือง
3. โปรแกรมจะจับเวลาตั้งแต่เริ่มส่งคำขอจนได้ข้อมูลครบทั้งหมด
4. แสดงผลลัพธ์ข้อมูลสภาพอากาศและเปรียบเทียบเวลาที่ใช้ของแต่ละวิธี
